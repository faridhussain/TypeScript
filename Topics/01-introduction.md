# Introduction to TypeScript
## What is TypeScript?

TypeScript is an open-source programming language developed by `Microsoft`.

It is a superset of JavaScript, which means every valid JavaScript program is also valid TypeScript code.

TypeScript adds features on top of JavaScript such as:
- static typing
- interfaces
- enums
- generics
- better tooling and autocompletion

## Why was TypeScript created?
JavaScript was originally designed for small scripts in web pages.

As applications became larger, managing JavaScript code became more difficult because many errors could only be found while running the application.

TypeScript was created to solve this problem by detecting many errors during development before the code reaches production.

## Who created TypeScript?
TypeScript was created by `Anders Hejlsberg` and was first released by Microsoft in 2012.

**Anders Hejlsberg** is also known for creating the Pascal language and being one of the lead architects of `C#`.

## JavaScript VS TypeScript

**JavaScript:**
- dynamically typed
- errors are often discovered at runtime
- no built-in type system

**TypeScript:**
- statically typed
- catches many errors during development
- provides a powerful type system
- improves code maintainability

## How TypeScript Works?

Browsers do not understand TypeScript directly.

TypeScript code is first compiled into JavaScript.

typescript (.ts) -> typescript compiler -> javascript (.js) -> browser

## Why learn TypeScript
- helps catch bugs earlier
- improves developer experience
- makes large codebases easier to maintain
- widely used in modern React applications
- commonly required in professional development
